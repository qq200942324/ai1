## 实战代码

### 一、腾讯接口 —— 零依赖快速看盘

```python
import requests

codes = ['sh000001', 'sz399001', 'sz399006', 'sh600519']
url = f"https://web.sqt.gtimg.cn/q={','.join(codes)}"
headers = {'User-Agent': 'Mozilla/5.0', 'Referer': 'https://gu.qq.com/'}
resp = requests.get(url, headers=headers)
resp.encoding = 'gbk'

for line in resp.text.split(';'):
    if '~' in line:
        p = line.split('~')
        print(f"{p[1]:8s} {p[3]:>8s}  {p[32]:>6s}%")
```

**代码格式**：`sh` = 上海，`sz` = 深圳（上海主板和科创板都用 `sh`）

### 二、东方财富 —— 涨幅榜 & 板块资金流

```python
import requests

# 涨幅榜前 20
url = "https://push2delay.eastmoney.com/api/qt/clist/get"
params = {
    "fid": "f3", "po": 1, "pz": 20,
    "fs": "m:0+t:6,m:0+t:80,m:1+t:2,m:1+t:23",
    "fields": "f2,f3,f12,f14,f15,f16,f17"
}
data = requests.get(url, params=params).json()
for item in data['data']['diff']:
    print(f"{item['f12']} {item['f14']:8s} "
          f"¥{item['f2']:>8.2f} {item['f3']:>+7.2f}% "
          f"换手{item['f15']}% 量比{item['f17']}")
```

**常用 `fs` 参数**：

| 参数值 | 含义 |
|--------|------|
| `m:0+t:6,m:0+t:80` | 沪深 A 股 |
| `m:0+t:13` | 创业板 |
| `m:1+t:23` | 科创板 |
| `m:90+t:2` | 行业板块 |

#### 东方财富新闻 API

```python
import requests

# 市场热点新闻
url = "https://np-listapi.eastmoney.com/comm/web/getNewsByColumns"
params = {
    "client": "web",
    "columnId": "102",       # 102=市场热点, 101=头条, 106=行业
    "pageNo": 1,
    "pageSize": 20
}
data = requests.get(url, params=params).json()
for item in data['data']['list']:
    print(f"{item['showDate']}  {item['title']}")
```

## 重要注意事项

| 事项 | 说明 |
|------|------|
| **请求频率** | 建议 ≥ 3 秒/次，太频繁会被封 IP |
| **编码** | 腾讯接口返回 GBK，需 `.encoding = 'gbk'` |
| **接口变更** | 东方财富可能改域名（已验证延时域名 `push2delay`） |
| **优先用库** | 裸调 HTTP 随时可能失效，优先用 AkShare/mootdx |
| **本地缓存** | 历史数据拉一次就缓存，别反复请求 |

## 与 Wiki 交易框架的结合

```
抓行情数据 → 抓新闻消息面
       ↘     ↙
     形成完整盘面认知
        ↓
用 [[情绪周期]] 判断当前阶段
        ↓
用 [[主要矛盾与核心论]] 锁定核心方向
        ↓
用 [[打板低吸半路]] 匹配进场手法
        ↓
用 [[仓位管理]] 决定下注比例
```

> **核心原则**：量价数据告诉你「正在发生什么」，消息面告诉你「为什么发生」以及「市场在交易什么预期」。二者缺一不可。

## 七、20 日滚动数据库（自动化管线）

> 每次启动会话时自动补齐近 20 日行情 + 新闻数据，删除过期旧数据。数据存储在 `2-wiki/data/` 中。

### 运行方式

```bash
# 在 vault 根目录运行
python scripts/update_data.py
```

**LLM 会话启动时**，检查 `2-wiki/data/meta.json` 中的 `last_update` 字段，如果超过 1 天未更新则自动执行脚本。

### 数据存储结构

```
2-wiki/data/
├── meta.json                          # 元数据：最后更新时间、窗口范围
├── market/
│   ├── indices_master.json            # 8大指数×20日 K线（JSON）
│   ├── indices_by_date.json           # 按日期查询（JSON）
│   └── market_summary.md              # 人类可读摘要（Obsidian 友好）
└── news/
    ├── news_master.json               # 7830+ 条新闻（JSON）
    └── news_summary.md                # 按日期+分类的摘要（Obsidian 友好）
```

### 数据源

| 数据类型 | 来源 | 覆盖 |
|---------|------|------|
| **行情K线** | 腾讯 K线 API (`web.ifzq.gtimg.cn`) | 8 大指数：上证/深证/创业板/科创50/上证50/沪深300/中证500/中证1000 |
| **新闻消息** | 新浪财经滚动新闻 API (`feed.mix.sina.com.cn`) | 5 大分类：A股沪深/宏观政策/公司要闻/国际财经/行业板块 |

### 数据字段

**行情**：`date`, `open`, `close`, `high`, `low`, `volume`, `amount`(当日), `change_pct`, `amplitude`
**新闻**：`title`, `url`, `date`, `time`, `keywords`, `category`

### 20 日滚动机制

- 每次运行只抓取窗口期内的缺失日期
- 自动删除 `date < (today - 20days)` 的旧数据
- 新闻去重（按 URL）
- 每日新闻量 200-2000 条不等，总量维持在 7000+ 条

# LLM Wiki 通用操作手册

> **给任何 AI 看**：读完后你就能像专业的知识管理员一样操作这个 Obsidian Wiki。
> 
> 理念来源：Andrej Karpathy 的 [LLM Wiki 模式](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)
>
> **核心比喻**：Obsidian 是 IDE，AI 是程序员，Wiki 是代码库。人类负责收集资料、提出问题；AI 负责所有整理、摘要、交叉引用和维护工作。

---

## 系统概述

这是一个 Obsidian 知识库（Vault），位于 `D:\happy\`。它不是一个普通的笔记文件夹——它是一个**由 AI 维护的 Wiki 系统**。

### 三层架构

```
┌──────────────────────────────────────────┐
│  0-inbox/ + 1-sources/  ← 原始资料层      │  只读，真相来源（人类投放）
├──────────────────────────────────────────┤
│  2-wiki/                ← Wiki 层         │  AI 生成 & 维护
├──────────────────────────────────────────┤
│  AI-MANUAL.md / CLAUDE.md ← Schema 层    │  AI 操作规范（你正在读的）
└──────────────────────────────────────────┘
```

### 关键原则

- **原始资料层只读**：`0-inbox/` 和 `1-sources/` 中的文件是"真相来源"，AI 只读取不修改
- **Wiki 层由 AI 维护**：`2-wiki/` 中的每个页面都是 AI 生成的，人类不直接编辑
- **每次操作都记录**：`2-wiki/log.md` 按时间线记录每一次变更
- **链接是第一公民**：`[[Wiki-link]]` 让知识之间产生关联，图视图可视化

## 核心操作

### 操作 1：Ingest（摄入新资料）

**触发**：人类把资料放入 `0-inbox/` 后说「处理 inbox」「ingest」「消化」

**流程**（7 步，缺一不可）：

```
1. 读取 → 阅读 0-inbox/ 中的新资料
2. 讨论 → 向人类简要总结关键发现
3. 写摘要 → 在 2-wiki/summaries/ 创建「[资料名].md」
4. 提取概念 → 在 2-wiki/concepts/ 创建或更新相关页面
5. 更新索引 → 在 2-wiki/index.md 添加或更新条目
6. 追加日志 → 在 2-wiki/log.md 追加本次操作记录
7. 归档 → 将原始资料从 0-inbox/ 移入 1-sources/
```

**命名规范**：
- 概念页面：`[概念名].md`（如 `情绪周期.md`）
- 摘要页面：`[来源名] - 摘要.md`
- 综合页面：描述性标题（如 `课程精华-跨主题深度框架.md`）

### 操作 2：Query（回答提问）

**触发**：人类直接提问

**流程**：
1. 搜索 `2-wiki/` 中相关页面（概念、摘要、综合）
2. 综合答案，必须附带引用（用 `[[页面名]]` 格式链接到具体 Wiki 页面）
3. 如果答案有价值，问人类「要不要把这个答案归档成新页面？」

### 操作 3：Lint（健康检查）

**触发**：人类说「lint」「检查 Wiki」「健康检查」

**检查项**：
- 矛盾声明（不同页面对同一概念给出矛盾描述）
- 孤立页面（无入链的页面，没有任何 `[[xxx]]` 指向它）
- 过时信息（`updated` 日期超过 3 个月的页面）
- 缺失概念（被频繁提及但还没有独立页面的概念）
- 交叉引用缺口（语义相关但没有互相链接的页面）

### 操作 4：组织与维护

**触发**：人类说「整理」「合并」「拆分」「导出」等

- 合并页面：去重合并，更新所有 `[[引用]]`
- 拆分页面：按子主题拆成多个页面
- 批量打标签：更新 frontmatter 中的 `tags:` 字段
- 导出：合并多个 Wiki 页面为长文输出到 `3-outputs/`

## 所有 Wiki 页面的统一格式

### 必须包含 YAML frontmatter

```yaml
---
tags: [标签1, 标签2]      # Obsidian 标签系统
created: 2026-05-27       # 创建日期
updated: 2026-05-27       # 最后更新日期
sources:                  # 溯源链（引用了哪些原始资料）
  - "[[1-sources/xxx]]"
---
```

### 链接规范

- **一律使用 Wiki-link**：`[[概念名]]` 或 `[[2-wiki/concepts/概念名]]`
- 每个页面中**首次出现**某个概念时必须建立链接
- 每个 Wiki 页面**至少被另一个页面引用**（杜绝孤立页面）
- 正文中引用其他页面用 `[[概念名]]`，引用原始资料用 `[[1-sources/xxx]]`

### index.md 条目格式

按分类分组，使用表格：

```markdown
## 概念 (concepts)

| 页面 | 摘要 | 日期 | 来源数 |
|------|------|------|--------|
| [[2-wiki/concepts/xxx]] | 一句话描述 | 2026-05-27 | 3 |
```

### log.md 条目格式

只追加，不修改历史。每条日志使用统一前缀：

```markdown
## [2026-05-27] ingest | 资料标题
- 创建摘要：[[2-wiki/summaries/xxx]]
- 更新概念：[[2-wiki/concepts/yyy]]（新增对比表）
```

操作类型：`ingest`（摄入）、`query`（回答）、`lint`（检查）、`setup`（配置）、`convert`（格式转换）

## 输出规范（给 AI 看）

当 AI 生成 Wiki 内容时，遵循以下规范：

1. **中文为主**，专有名词保留原文
2. **引用必须有出处**：每个结论链接到具体的来源页面
3. **表格优先于段落**：对比、分类、步骤类信息优先用表格呈现
4. **一句话摘要**：每个页面开篇有一个 `> blockquote` 摘要
5. **适度使用 Mermaid**：流程图、关系图用 Mermaid 代码块
6. **每页至少 3 个链接**：提高知识网络密度
7. **不凭空断言**：不确定的内容标注「待验证」或追问人类

## 快速开始（给人类看）

### 怎么让其他 AI 使用这个系统？

**方法一：复制粘贴本文件**

1. 打开 ChatGPT / Gemini / 其他 AI 的新对话
2. 将本文件 `AI-MANUAL.md` 的**全部内容**粘贴进去
3. 然后加上你的指令，例如：
   ```
   以上是我的 Wiki 系统。请处理 0-inbox 中新增的三篇资料。
   ```
   或
   ```
   以上是我的 Wiki 系统。请基于我的 [[情绪周期]] 和 [[主要矛盾与核心论]] 框架，
   分析当前市场状态并给出建议。
   ```

4. AI 会按照手册规范输出符合格式的内容，你将其复制到对应的 `.md` 文件中

**方法二：作为 system prompt**

如果你用的 AI 工具支持 system prompt（如 Claude API、OpenAI API），将本文件内容作为 system prompt 设置，然后正常对话即可。

**方法三：用支持文件系统的 AI 工具**

Claude Code、Cursor、Windsurf 等工具可以直接读取整个 Vault：
- 将 `AI-MANUAL.md` 作为项目 context 文件
- AI 启动后自动按手册规范操作

### 常用指令速查

| 对 AI 说的 | 效果 |
|-----------|------|
| `处理 inbox` / `ingest` | 消化 inbox 中所有新资料 |
| `处理 [文件名]` | 只消化指定资料 |
| `什么是 [概念]？` | 搜索 Wiki 综合答案 |
| `对比 A 和 B` | 跨页面对比分析 |
| `lint` / `检查 Wiki` | 全面健康检查 |
| `把这个答案归档` | 将当前答案保存为新 Wiki 页面 |
